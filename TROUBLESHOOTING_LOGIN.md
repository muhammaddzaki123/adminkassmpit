# Troubleshooting: "FATAL: Tenant or user not found" Login Error

## Problem
Student login fails with error: `FATAL: Tenant or user not found`

## Root Causes & Solutions

### 1. ✅ FIXED: Missing JWT_SECRET (Primary Cause)
**Issue**: `JWT_SECRET` was not set in `.env`, causing the app to use default value `'your-secret-key'`

**Status**: ✅ Fixed in `.env`
- Added: `JWT_SECRET=your-super-secret-jwt-key-change-in-production-12345`
- This enables proper token signing/verification

**Action**: Restart the application after this change

---

### 2. Database Connection Pool Issue (Supabase)
**Issue**: Error "Tenant or user not found" from PostgreSQL pgbouncer

**Diagnosis**:
```bash
# Check if database connection is working
# Try this when you get the error
```

**Possible Causes**:
- Connection pooler authentication failure
- RLS (Row Level Security) policies preventing access
- Invalid connection string format

**Solutions**:
- ✅ Verify `DATABASE_URL` and `DIRECT_URL` in `.env`
- ✅ Both URLs should have matching credentials
- ✅ Check Supabase dashboard for any connection issues

---

### 3. Student User Account Not Created
**Issue**: Admin created student data, but user account wasn't created

**How to Create Student Properly**:
1. Go Admin Panel → Students → Add Student
2. Fill in form (Nama, NISN, Kelas, etc.)
3. Submit form
4. System will return:
   - Username: `{NISN}_student` (or custom)
   - Password: `{NISN}` (default) or custom
5. **Save these credentials** - student needs them to login

**If Student was Created Wrong**:
- ✅ Use endpoint: POST `/api/admin/students/create`
- This creates BOTH Student + User records in ONE transaction
- Returns credentials immediately

- bcrypt salt rounds: 10
- Hash generated by: `bcrypt.hash('1234567890', 10)`

## Supabase RLS Note

If `FATAL: Tenant or user not found` persists after fixes:

**Possible RLS Issues** (advanced configuration):
- Check if Row-Level Security is enabled on `users` table
- Go to Supabase Dashboard → SQL Editor
- Run: `SELECT pg_catalog.obj_description(oid) FROM pg_class WHERE relname = 'users'`
- If RLS policies exist, verify they allow SELECT for your role
